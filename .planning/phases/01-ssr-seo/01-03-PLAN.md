<objective>
Replace the hand-maintained `src/sitemap.xml` with an auto-generated file produced by a build script that reads the canonical route list from `routes.txt` (written in Plan 01-01) plus the blog post slugs from `src/app/features/blog/blog-content.ts`. The script runs before every production build so the sitemap is always in sync with what is actually prerendered.

Purpose: Currently `src/sitemap.xml` is hand-edited and drifts from reality â€” for example it lists `/pricing` (which is actually routed as `/home/pricing`) and would need manual updates every time a route is added or a blog post is published. SEO-02 requires sitemap generation from the route config rather than manual maintenance.

Output: A `scripts/generate-sitemap.mjs` ES module script, a `prebuild-production` npm script that invokes it, and a regenerated `src/sitemap.xml` that matches routes.txt + blog slugs. After this plan, editing `routes.txt` or adding a blog post to `BLOG_POSTS` is the only change needed to ship a new sitemap entry on the next deploy.
</objective>
