<objective>
Replace the scattered per-component `Title`/`Meta` calls with a centralized `SeoService` that sets title, meta description, canonical URL, Open Graph tags, Twitter Card tags, and optional page-specific JSON-LD structured data in one call. Wire every public route's component to use the service. This delivers SEO-01 (unique per-route titles/descriptions/OG tags) and SEO-03 (page-specific JSON-LD for home, pricing, tools, and blog).

Purpose: After 01-01, Google crawls prerendered HTML — but every page currently shares the same `<title>Band Central</title>` from `index.html`, and only a few pages (home, tools) have ad-hoc Meta service calls. Rich search snippets require per-page titles, descriptions, and structured data. Without this, Google indexes the pages but treats them all as duplicates of the home page.

Output: A single `SeoService` consumed by every public route component, emitting unique title + description + OG tags + (where appropriate) JSON-LD blocks in the prerendered HTML. The existing global Organization + SoftwareApplication JSON-LD in `src/index.html` remains unchanged — page-specific JSON-LD is added alongside, not in place of.
</objective>

**Implementation details:**
- Inject `Title`, `Meta`, `@Inject(DOCUMENT) doc: Document`, `@Inject(PLATFORM_ID) platformId`
- For canonical link: `this.doc.querySelector('link[rel="canonical"]')` — if null, create via `this.doc.createElement('link')`, set `rel` and `href`, append to `this.doc.head`; if present, update `href`.
- For JSON-LD script: same create-or-update pattern with `this.doc.getElementById(id)`. When creating, set `type = 'application/ld+json'` and assign `textContent = JSON.stringify(schema)`. When updating, just reassign `textContent`.
- `setSeo` should internally call `this.meta.updateTag({ property: 'og:title', content: ... })` etc. — Angular's Meta service handles create-or-update based on the attribute selector it infers from the passed object. For twitter tags use `{ name: 'twitter:card', content: ... }` (twitter uses `name`, not `property`).
- Do NOT use `document` or `window` globals directly — always go through `this.doc`.
- The service must be safe to call during prerender: `setJsonLd` running on the server should add the script to the server-rendered DOM so it appears in the prerendered HTML output. Do NOT wrap the body in `isPlatformBrowser` — that would prevent the SEO tags from being prerendered, defeating the entire purpose.
