Fix every module-definition-time or bootstrap-time reference to `window`/`document`/`localStorage` so `ng build --configuration=production` completes without `ReferenceError: window is not defined` or hanging. The runtime @HostListener handlers in header.component.ts, tap-tempo.component.ts, etc., are fine as-is because they only execute in the browser — focus on code that runs during Angular's server-side module initialization and constructor execution.

**File 1 — src/app/shared/helpers/window.helper.ts (COMPLETE REWRITE):**
Replace the current eager `_window()` factory with an SSR-safe version. The new file should export:
```typescript
import { InjectionToken, PLATFORM_ID, inject } from '@angular/core';
import { isPlatformBrowser } from '@angular/common';

export const WINDOW = new InjectionToken<Window | null>('WindowToken', {
  factory: () => isPlatformBrowser(inject(PLATFORM_ID)) ? window : null
});
```
Preserve the `WindowRef`/`BrowserWindowRef`/`WINDOW_PROVIDERS` exports for backwards compatibility with `shared.module.ts` — but rewrite `_window()` to `() => (typeof window !== 'undefined' ? window : null)` and update the class provider usage so nothing evaluates at import time. The simplest approach: keep `WINDOW_PROVIDERS` as an empty array (the token now self-provides via its factory) and keep `BrowserWindowRef` for any class-based consumer, gated with a `typeof window !== 'undefined'` check inside `nativeWindow`.

Update `src/app/features/home/header/header.component.ts` only if its `@Inject(WINDOW) private window: Window` declaration needs to change to `Window | null`; then guard the `onWindowScroll` method's `this.window.pageYOffset` access with an `if (!this.window) return;` early exit. (The existing handler only fires on scroll so it naturally only runs in the browser, but the type needs to allow null.)

**File 2 — src/app/core/core.module.ts (FIX LINE 25):**
The line `{ provide: 'LOCALSTORAGE', useValue: window.localStorage }` evaluates `window` at module-definition time and will CRASH the prerender build on the very first module import. Replace with a factory provider:
```typescript
import { PLATFORM_ID } from '@angular/core';
import { isPlatformBrowser } from '@angular/common';
// ...
{
  provide: 'LOCALSTORAGE',
  useFactory: (platformId: object) => isPlatformBrowser(platformId) ? window.localStorage : null,
  deps: [PLATFORM_ID]
}
```
Also grep the codebase for `inject('LOCALSTORAGE')` or `@Inject('LOCALSTORAGE')` usages; if any consumer dereferences the value without null-checking, leave a TODO comment there but do not fix in this task (those features run browser-only and will be addressed if prerender surfaces them).

**File 3 — src/app/app.component.ts (GUARD AUTH REDIRECT):**
Per RESEARCH.md Pitfall 5, the `router.events.subscribe` in the constructor runs during prerender for every route since AppComponent wraps `<router-outlet>`. Subscribing to `authService.isLoggedIn$` during prerender may trigger Firebase Auth initialization and corrupt the prerendered output with a redirect. Guard the entire subscription:
```typescript
import { Component, PLATFORM_ID, inject } from '@angular/core';
import { isPlatformBrowser } from '@angular/common';
// In constructor:
constructor(authService: AuthenticationService, router: Router) {
  if (isPlatformBrowser(inject(PLATFORM_ID))) {
    router.events.subscribe((event: any) => { /* existing logic unchanged */ });
  }
}
```

**File 4 — src/app/shared/directives/scroll-to.directive.ts (GUARD METHODS):**
The `currentYPosition()` and `elmYPosition()` methods reference `self.pageYOffset`, `document.documentElement`, `document.body`, and `document.getElementById` unconditionally. Though these only run on click (safe in browser), the class is imported into bundles that may be evaluated during prerender. Inject `PLATFORM_ID` and add an `isPlatformBrowser(this.platformId)` early-return at the top of `smoothScroll()`, `currentYPosition()`, and `elmYPosition()`. Also replace the two `setTimeout("window.scrollTo(0, " + leapY + ")", ...)` string-based timeouts with arrow functions: `setTimeout(() => window.scrollTo(0, leapY), timer * speed)` (fixes a longstanding string-eval bug, and string `setTimeout` would also fail in strict CSP environments).

**File 5 — src/app/features/home/home/home.component.ts (GUARD ngAfterViewInit):**
Line 41 calls `document.getElementById(fragment)` inside `ngAfterViewInit`. `ngAfterViewInit` does not run during prerender hydration output, but guard it anyway as defense-in-depth:
```typescript
import { PLATFORM_ID, inject } from '@angular/core';
import { isPlatformBrowser } from '@angular/common';
// In class:
private platformId = inject(PLATFORM_ID);
ngAfterViewInit() {
  if (!isPlatformBrowser(this.platformId)) return;
  this.route.fragment.subscribe(fragment => { /* existing */ });
}
```
